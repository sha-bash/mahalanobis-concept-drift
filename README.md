# Программный комплекс «Mahalanobis Concept Drift Detector»

Программный комплекс предназначен для классификации текстовых данных
и обнаружения концептуального дрейфа на основе многомодальной метрики
Махаланобиса в пространстве эмбеддингов трансформерных моделей.

Система применяется для анализа пользовательских обращений,
мониторинга изменений в текстовых потоках и контроля устойчивости
NLP-моделей.

---

## 1. Назначение программы

Программа реализует следующие функции:

- преобразование текстов в векторные представления (эмбеддинги);
- формирование многомодальной модели нормы по заданным кластерам;
- вычисление расстояния Махаланобиса до центров кластеров;
- автоматическое определение порога отклонения;
- детекцию концептуального дрейфа;
- сохранение и загрузку обученных моделей;
- работу через интерфейс командной строки (CLI);
- работу через демонстрационное веб-приложение (Streamlit).

---

## 2. Принцип работы

Для каждого семантического кластера вычисляются:

- центр распределения (средний вектор эмбеддингов);
- ковариационная матрица признакового пространства;
- адаптивный порог на основе квантиля расстояний.

Для нового текста определяется минимальное расстояние Махаланобиса
до одного из кластеров.

Если расстояние превышает соответствующий порог,
фиксируется наличие концептуального дрейфа.

---

## 3. Форматы входных и выходных данных

### 3.1 Входные данные

Программа принимает на вход текстовые данные в формате CSV.

#### Обязательные параметры:

- `text_columns` — список колонок, содержащих текст;
- `label_column` — колонка, содержащая идентификатор кластера.

#### Пример обучающего CSV-файла:

```csv
subject,body,queue
"Ошибка при оплате","Не проходит платёж по карте","Billing"
"Проблема со входом","Не могу авторизоваться в системе","Technical Support"
```

### Описание полей

- **subject** — краткое описание обращения;
- **body** — подробный текст обращения;
- **queue** — семантический кластер (метка класса).

Допускается использование иных названий колонок при корректной передаче параметров
через интерфейс командной строки или веб-интерфейс.

---

### 3.2 Входные параметры CLI

Основные параметры запуска:

- `--data` — путь к CSV-файлу или ZIP-архиву с данными;
- `--label-column` — имя колонки с метками;
- `--text-columns` — список текстовых колонок;
- `--threshold-quantile` — квантиль для расчёта порога;
- `--min-cluster-size` — минимальный размер кластера;
- `--model-file` — путь к сохранённой модели;
- `--text` — текст для анализа в режиме предсказания.

---

### 3.3 Выходные данные

Для каждого анализируемого текста формируется следующий набор значений:

- `predicted_label` — предсказанный кластер;
- `distance` — минимальное расстояние Махаланобиса;
- `threshold` — пороговое значение для соответствующего кластера;
- `score` — разность *(distance − threshold)*;
- `drift` — булев флаг наличия концептуального дрейфа.

---

### 3.4 Результаты режима оценки (eval)

В режиме оценки дополнительно формируются:

- файл `predictions.csv` — детальные предсказания по каждому примеру;
- файл `metrics.json` — агрегированные метрики качества:
  - `precision`,
  - `recall`,
  - `F1-score`,
  - `accuracy`,
  - `ROC-AUC`,
  - матрица ошибок.

Результаты сохраняются в каталог `reports/`.
