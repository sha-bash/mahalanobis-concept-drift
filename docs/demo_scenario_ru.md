# ДЕМОНСТРАЦИОННЫЙ СЦЕНАРИЙ РАБОТЫ ПРОГРАММЫ

## 1. Назначение демонстрационного сценария

Демонстрационный сценарий предназначен для наглядной иллюстрации работы программного комплекса «Mahalanobis Concept Drift Detector», реализующего классификацию текстовых данных и обнаружение концептуального дрейфа на основе многомодальной метрики Махаланобиса в пространстве эмбеддингов трансформерных моделей.

Сценарий демонстрирует:

- построение эмбеддингов текстовых данных;
- формирование многомодальной статистической модели нормы;
- автоматическую детекцию концептуального дрейфа;
- количественную оценку качества детекции;
- визуализацию результатов анализа.

Сценарий предназначен для демонстрации работы программы экспертам и членам комиссии при регистрации программного продукта.

---

## 2. Используемый набор данных

В качестве демонстрационного набора данных используется открытый датасет  
**Multilingual Customer Support Tickets (Kaggle)**, содержащий обращения пользователей в службу поддержки.

Основные поля датасета:

- `subject` — краткое описание обращения;
- `body` — текст обращения;
- `queue` — категория или очередь обработки (семантический кластер).

В рамках демонстрации:

- данные объединяются из полей `subject` и `body`;
- поле `queue` используется как идентификатор семантического кластера.

Разделение выполняется по кластерам:

- часть кластеров используется для обучения модели;
- оставшиеся кластеры используются для демонстрации концептуального дрейфа (Out-of-Distribution).

Такой подход обеспечивает корректную демонстрацию появления новых семантических контекстов, отсутствующих на этапе обучения.

---

## 3. Общая логика демонстрационного сценария

Демонстрационный сценарий представляет собой сквозной эксперимент, включающий следующие этапы:

1. загрузка и предварительная обработка данных;
2. построение эмбеддингов текстов;
3. формирование статистической модели нормы;
4. детекция концептуального дрейфа;
5. вычисление метрик качества;
6. визуализация результатов.

Каждый этап может быть воспроизведён через интерфейс командной строки или демонстрационное веб-приложение.

---

## 4. Этапы демонстрационного сценария

### 4.1. Подготовка и разбиение данных

Данные загружаются из CSV или ZIP-архива.

Производится:

- удаление некорректных записей;
- объединение текстовых полей;
- разбиение кластеров на:
  - обучающие (IN-distribution);
  - отложенные (OOD).

Кластеры, размер которых меньше установленного порога, исключаются из анализа.

---

### 4.2. Формирование эмбеддингов

Для текстов обучающей выборки вычисляются эмбеддинги с использованием модели SBERT.

Эмбеддинги представляют тексты в виде векторов фиксированной размерности в семантическом пространстве.

---

### 4.3. Построение многомодальной модели нормы

Для каждого обучающего кластера вычисляются:

- средний вектор эмбеддингов;
- ковариационная матрица;
- адаптивный порог на основе заданного квантиля расстояний Махаланобиса.

Совокупность этих параметров образует многомодальную статистическую модель нормы.

---

### 4.4. Детекция концептуального дрейфа

Для каждого нового текста:

1. вычисляется эмбеддинг;
2. рассчитывается расстояние Махаланобиса до всех известных кластеров;
3. выбирается минимальное расстояние;
4. выполняется сравнение с соответствующим порогом.

Если расстояние превышает порог, фиксируется концептуальный дрейф.

---

### 4.5. Расчёт метрик качества

В режиме `eval` дополнительно рассчитываются:

- precision;
- recall;
- F1-score;
- accuracy;
- ROC-AUC;
- матрица ошибок.

Метрики рассчитываются для задачи детекции дрейфа, где:

- IN-кластеры считаются нормой;
- OOD-кластеры считаются дрейфом.

---

### 4.6. Визуализация результатов

В демонстрационном приложении выполняется:

- проекция эмбеддингов в двумерное пространство (PCA);
- отображение распределения расстояний;
- визуальное выделение наблюдений с дрейфом.

Визуализация повышает наглядность демонстрации и облегчает интерпретацию результатов.

---

## 5. Ожидаемые результаты демонстрации

В ходе демонстрации пользователь наблюдает:

- корректную классификацию текстов обучающих кластеров;
- увеличение расстояний Махаланобиса для новых кластеров;
- автоматическое обнаружение концептуального дрейфа;
- количественную оценку качества детекции;
- визуальное разделение известных и новых контекстов.

---

## 6. Применение демонстрационного сценария

Демонстрационный сценарий используется для:

- проверки корректности реализации алгоритма;
- демонстрации функциональных возможностей программы;
- использования в качестве демонстрационного стенда при регистрации программы для ЭВМ;
- обучения пользователей работе с системой.
