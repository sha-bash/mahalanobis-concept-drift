## 1. Назначение программы

Программа «Mahalanobis Concept Drift Detector» предназначена для автоматической детекции концептуального дрейфа в потоках текстовых данных на основе многомодальной метрики Махаланобиса в пространстве эмбеддингов трансформерных моделей.

Программа обеспечивает выявление статистически значимых отклонений входных данных от обученного распределения без дополнительного дообучения модели.

---

## 2. Функциональные возможности

Программа реализует следующие функции:

1. Преобразование текстовых сообщений в векторные представления фиксированной размерности.
2. Формирование статистической модели нормы для каждого семантического кластера.
3. Вычисление расстояния Махаланобиса до каждого кластера.
4. Определение ближайшего кластера.
5. Расчёт адаптивного порога детекции.
6. Определение наличия концептуального дрейфа.
7. Формирование числовой оценки степени отклонения.

Входные данные представляют собой набор текстовых сообщений с указанием идентификатора кластера.

---

## 5. Порядок работы программы

### 5.1 Этап построения модели нормы

1. Загрузка обучающего набора данных.
2. Преобразование текстов в эмбеддинги фиксированной размерности.
3. Для каждого кластера вычисляются:
   - средний вектор;
   - ковариационная матрица;
   - регуляризованная ковариационная матрица;
   - статистический порог детекции.

Результатом этапа является модель «нормального» распределения данных.

---

### 5.2 Этап детекции

Для нового текстового сообщения выполняются следующие действия:

1. Вычисляется эмбеддинг.
2. Рассчитываются расстояния Махаланобиса до всех кластеров.
3. Определяется ближайший кластер.
4. Вычисляется минимальное расстояние.
5. Выполняется сравнение с порогом.
6. При превышении порогового значения фиксируется наличие концептуального дрейфа.

---

## 6. Выходные данные

Результатом работы программы являются:

- идентификатор ближайшего кластера;
- минимальное расстояние Махаланобиса;
- значение порога детекции;
- бинарный индикатор наличия дрейфа;
- числовая оценка величины отклонения.

Пример вывода:

cluster_id: 2
distance: 11.84
threshold: 8.97
drift: True
score: 2.87


---

## 7. Ограничения и особенности

1. Для корректной работы требуется наличие обучающей выборки достаточного объёма.
2. При малом количестве данных возможна нестабильность ковариационных оценок.
3. Для высокоразмерных эмбеддингов обязательна регуляризация ковариационной матрицы.
4. Программа не выполняет автоматическое дообучение модели.

---

## 8. Обработка ошибок

В случае:

- отсутствия обучающих данных;
- некорректного формата входных данных;
- невозможности инверсии ковариационной матрицы;

программа формирует сообщение об ошибке с указанием причины.
